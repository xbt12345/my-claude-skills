# 任务诊断

## D1 行家现场

描述现实场景、判断材料、责任和取舍，不写装饰性头衔。

```text
场景：
判断对象：
可用材料与工具：
失败责任：
速度、质量、风险如何取舍：
```

检查：先看什么信号；哪些弱信号会警报；缺什么信息时拒绝下结论；什么做法表面专业但不会采用。

## D2 失败画像

每个失败模式必须回答：

```text
坏结果具体长什么样？
用户在哪个环节发现不能用？
原因属于事实、推理、取舍、表达还是执行接口？
哪条约束能提前阻断？
```

禁止只写“不深入、不专业、太泛、逻辑差”。

## D3 深层失败

依次检查：

1. 交付物完成，但没有推动真实目标。
2. 任务建立在未经验证或失效的前提上。
3. 优化容易测量的指标，牺牲真实价值。
4. 时间、预算、权限、受众、合规或工具能力缺失。
5. 用户表面要内容，实际要决策、说服、验证或行动。
6. 任务需要数据、权限、工具或现实反馈，不能由 Prompt 单独解决。

发现深层失败时，先修正任务定义。

## D4 最高优先级约束

```text
约束：
原因：
违反时的可观察结果：
执行中如何检查：
```

只能有一条。它必须改变执行取舍，并在冲突时覆盖篇幅、格式和风格要求。
