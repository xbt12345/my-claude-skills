# 领域模式库

> 仅在任务属于下列领域、且需要成熟结构时读取。这里提供可复用模块，不是完整 Prompt；每次仍须先确认 Goal、约束、失败模式与验收标准。

## 共用前置合同

```text
[真实目标] 任务完成后要改变的决策、能力、认知或行动
[操作模型] 现实工作场景、判断材料、风险责任与警觉信号
[最高约束] 冲突时覆盖格式、篇幅和表现技巧的唯一规则
[失败模式] 3 个表面合格但实际不可用的结果
[验收证据] 用户可以观察、测试或复现的通过条件
```

禁止直接继承示例中的受众、指标、风格和假设。领域模式必须按当前任务裁剪。

## AI 与技术分析

**触发条件**：技术趋势、模型能力、工具选型、路线投入判断。  
**不适用条件**：单一术语解释，或没有任何决策场景。

### 技术趋势与工具分析

```text
[背景] 技术、阶段、使用场景、可比较替代方案、证据截止日期
[任务] 判断该技术会如何改变当前决策或工作
[分析]
1. 能力边界：稳定能做什么，不能做什么
2. 对比基线：相对现有方案强弱在哪里
3. 落地条件：数据、工程、成本、权限、学习曲线
4. 反证：什么事实会推翻当前判断
[输出] 证据 → 推论 → 行动；区分事实、估计和未知
```

### 技术路线决策

```text
[支持论证] 给出最强的可验证支持证据
[反对论证] 逐项挑战时机、替代方案、隐藏成本和失败案例
[综合] 条件性结论、证据强度、关键假设、结论翻转信号
[行动] 在明确时间窗口内做什么、不做什么、先验证什么
```

## 认知学习与复习

**触发条件**：概念理解、文章消化、知识整理、备考计划。  
**不适用条件**：只需定义、翻译，或要求精确覆盖却没有材料。

### 概念深度理解

```text
[机制] 用具体因果链解释概念，不以换词复述定义
[例子] 一个正例、一个近似但错误的反例
[预测] 学习者必须用概念预测一个新现象
[边界] 明确失效条件和不适用场景
[连接] 与已有概念的相似点、关键区别和层级关系
[验收] 学习者能独立解释、区分、预测并迁移
```

### 长文或书籍消化

```text
[提炼] 一句话核心论点
[证据] 最强证据、证据缺口、最大漏洞
[迁移] 可应用领域与不可迁移边界
[原则] 情境 + 机制 + 结果 + 失效条件
[价值] 哪些内容具有长期价值，哪些依赖时代条件
```

### 备考型整理

```text
[材料边界] 教材、讲义、PPT、考试范围与可用时间
[分级] P0 必须掌握 / P1 高频应用 / P2 扩展理解
[输出] 每项包含答题级定义、常见考法、至少一道检索或应用题
[计划] 按依赖和遗忘曲线分配复习、练习、回忆与模拟
[验收] 每个 P0 有测试；每日计划有明确完成证据
```

## 内容写作

**触发条件**：撰写、重写或优化文章、标题、开头和叙事结构。  
**不适用条件**：纯格式转换、摘要或逐字保留。

### 长文写作

```text
[目标变化] 读者读完后应理解、相信、感受或执行什么
[受众] 已知水平、关切、阻力与阅读场景
[核心命题] 一句话，可被反驳
[结构] 问题 → 洞见 → 证据/案例 → 边界 → 行动或余韵
[声音] 从用户样本提取句长、密度、态度、幽默和论证方式
[禁止] 列出具体套话、空结论和违背品牌声音的表达
[验收] 读者能复述核心命题，并知道下一步行动或判断
```

不要直接模仿创作者姓名；先把参考样本翻译成可观察的写作轴。

### 标题与开头

```text
[材料] 原标题、开头、正文命题、目标读者
[候选] 生成差异明确的标题方向，不批量同义改写
[约束] 标题承诺必须由正文兑现；禁止虚假悬念和垃圾词
[开头] 立即建立具体场景、冲突、信息缺口或读者代价
[验收] 读者能判断文章与自己何关，且预期与正文一致
```

## 投资与决策分析

**触发条件**：标的研究、行业机会、买卖持有或资源配置决策。  
**不适用条件**：要求保证收益、精确价格预测或替代持牌意见。

高风险任务必须补充期限、仓位、风险承受力、证据日期和可接受损失。

### 机会分析

```text
[决策] 入场、退出、持有还是继续观察
[基线] 当前价格/估值、替代资产、机会成本、时间窗口
[逻辑] 驱动力、现金流或价值来源、催化剂
[风险] 永久损失、逻辑失效、流动性和尾部风险
[反证] 哪些新证据会降低或推翻判断
[输出] 条件性行动，不把观点包装成确定预测
```

### 多空对抗

```text
[多方] 最强证据、估值安全边际、催化剂
[空方] 逐项反驳、隐藏假设、替代解释
[综合] 决策、证据强度、最脆弱假设、更新信号
[风险控制] 仓位上限、退出条件、最大可接受损失
```

## 项目、系统与自动化规划

**触发条件**：任务拆解、产品规划、系统架构、自动化流程、知识库或 Skill 优化。  
**不适用条件**：方案已经批准且用户只要求执行。

### Sprint 任务拆解

```text
[完成定义] 可运行、可观察或可复现的最终状态
[非目标] 本轮明确不做什么
[子任务] 每项只有一个责任，并附完成标准
[依赖] 串行项、并行项、外部依赖与人工确认点
[风险] 最可能失败位置、提前信号、降级路径
[验收] 命令、检查表、样例或用户路径
```

### 系统或产品方案

```text
[用户结果] 用户最终能完成什么
[约束] 现有系统、技术栈、数据、权限、时间和迁移边界
[候选方案] 至少两种真实可行路径
[权衡] 复杂度、成本、可靠性、可逆性、运维和演进空间
[推荐] 理由、关键假设和放弃条件
[实施] 分阶段、检查点、回滚与验收
```

### Bug 修复与体验改造混合任务

```text
[阶段 A] 先修可复现回归；每个问题独立验证
[阶段 B] 再确认审美方向和体验方案；不得与回归修复混交
[事实源] 开始前读取用户当前代码或配置，不按旧记忆修改
[验证] 自动测试 + 真实用户路径；记录性能和行为基线
```

### 文档库或知识库重组

```text
[盘点] 文件职责、链接、重复、历史、生产资产和单一真源
[合并] 只合并职责相同且生命周期一致的内容
[保留] 独立生产资产保持最小可检索单元
[顺序] 备份 → 合并 → 更新链接 → 删除旧文件 → 断链验收
[验收] 旧名称零引用、无断链、无空文件、无内容丢失
```

## 使用原则

1. 先选领域，再选一个主模式；不要把所有模块堆入 Prompt。
2. 模式只补领域判断，不替代 `Goal → Diagnose → Compile → Evaluate`。
3. 示例中的数量、时限和阈值必须由当前任务证据决定。
4. 高风险领域优先保留反证、失败处理和人工确认点。
5. 输出变长但行为没有改变时，删除该段。
