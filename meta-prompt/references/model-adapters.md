# Model Adapters

## ChatGPT / OpenAI

- 用清晰 section 和 JSON Schema 表达结构化输出。
- 将稳定规则放 system，将任务材料放 user。
- 工具与结构化输出能力可用时，不用自然语言模拟。

## Claude

- 长上下文使用 XML 风格区块分隔背景、规则、材料和交付。
- 把证据边界、工具权限和最终检查写成独立区块。
- 不要求展示隐藏推理。

## Gemini

- 使用明确标题、顺序动作和多模态材料说明。
- 指定材料与指令的边界，避免附件内容被当成命令。

## Image

- Prompt 聚焦可见结果：场景、构图、光线、材质、时间、排除项。
- Taste Contract 优先于空风格标签。
- 不虚构艺术总监履历，不堆叠互相冲突的风格名。

## Coding Agent

- 编译目标、范围、文件、工具权限、检查点、测试和停止条件。
- 不可逆动作前人工确认。
- 状态写入文件或 trace，不依赖长对话记忆。
- 完成声明必须绑定新鲜验证证据。
